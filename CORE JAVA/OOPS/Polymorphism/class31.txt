If we declare any method as final then overriding of that method is not possible.

ex:
---
class Parent 
{
	public final void property()
	{
		System.out.println("House - Not For Sale");
	}
}
class Child extends Parent 
{
	public void property()
	{
		System.out.println("House - For Sale");
	}
}
class  Test
{
	public static void main(String[] args) 
	{
		Parent p = new Child();
		p.property();
	}
}

Polymorphism 
=============
Polymorphism has taken from Greek word.

Here poly means many and morphism means forms.

The ability to represent in different forms is called polymorphism.

The main objective of polymorphism is to provide flexibility.

Diagram: class31.1

In java polymorphism, divided into two types.

1) Compile time polymorphism
--------------------------
A polymorphism which exhibits at compile time is called compile time polymorphism.
Method resolution will taken care by a compiler based on reference type.
A compile time polymorphism is also known as static polymorphism or early binding.
ex:
	Method Overloading 
	Method Hiding 

2) Runtime polymorphism 
------------------------
A polymorphism which exhibits at runtime is called runtime polymorphism.
Method resolution will taken care by a JVM based on runtime object.
A runtime polymorphism is also known as dynamic polymorphism or late binding.
ex:
	Method Overriding 


Sealed class
=============
A sealed class introduced in Java 17.

It provide a mechanism to restrict which other classes can't extend or implement a given class or interface. 

sealed modifier: 
----------------
The superclass or interface is declared with the sealed keyword.

permits clause:
---------------
The declaration must include a permits clause to explicitly list the direct subclasses or subinterfaces that are allowed to extend or implement it.


Every permitted subclass must be declared with one of three modifiers to define how the hierarchy continues:

final: 
------
The class cannot be extended any further (record classes are implicitly final).

sealed: 
-------
The class can be extended further, but only by its own specific permitted subclasses.

non-sealed: 
-----------
The class breaks the sealing hierarchy and can be extended by any unknown class (reverts to traditional open inheritance). 

ex:
----
sealed class A permits B
{
	public void m1()
	{
		System.out.println("M1-Method");
	}
}
//inheritence ends here 
final class B extends A 
{
	public void m2()
	{
		System.out.println("M2-Method");
	}
}
class Test 
{
	public static void main(String[] args)
	{
		A a = new A();
		a.m1();
		
		B b = new B();
		b.m1();
		b.m2();
	}
}
