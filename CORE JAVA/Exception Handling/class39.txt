finally block 
-------------
It is never recommanded to maintain cleanup code in try block because if any exception raised then it won't be executed.

It is never recommanded to maintain cleanup code in catch block because if there is no exception in try block then it won't be executed.

We need a place where we can maintain cleanup code and it should execute irrespective of exception raise or not . Such block is called finally block.

syntax:
------
	try 
	{
		- // Risky code 
	}
	catch(Exception e)
	{
		- // Error handling code 
	}
	finally	
	{
		- // Cleanup code 
	}

ex:
---
class Test 
{

	{
		try
		{
			System.out.println("try-block");
		}
		catch (Exception e)
		{
			e.printStackTrace();
		}
		finally
		{
			System.out.println("finally-block");
		}
	}
}

Q) What is the difference between final, finally and finalize method ?


final 
-----
It is a modifier which is applicable for variables, methods and classes.

If we declare any variable as final then reinitialization of that variable is not possible.

If we declare any method as final then overriding of that method is not possible.

If we declare any class as final then creating child class is not possible.  


finally
-------
It is a block which contains cleanup code and it should execute irrespective of exception raise or not. Such type of block is called finally block.


finalize
--------
It is a method called by garbage collector just before destroying an object for cleanup activity.


Garbage Collector 
=================
It is also known as deamon thread.

It is a leight weight thread which runs in a background to provide services.

It is a programming feature for memory allocation and deallocation.

There are two ways to call garbage collector in java.

1) System.gc() 

2) Runtime.getRuntime().gc()


1) System.gc()
--------------
class Test 
{
	//instance variable 
	int i=10;
	
	public static void main(String[] args) 
	{
		Test t = new Test();
		System.out.println(t.i); // 10
		
		t = null;
		System.gc();
	}
	public void finalize()
	{
		System.out.println("finalize method");
	}
}


2) Runtime.getRuntime().gc()
----------------------------
class Test 
{
	//instance variable

	public static void main(String[] args)

		t = null;
		Runtime.getRuntime().gc();
	}

		System.out.println("finalize method");
	}

try-with-resources in Java
==========================
A try-with-resources introduced in Java 7.

A try-with-resources is a try block which contains one or more resources.

A try-with-resources ensures that each rosource must be closed at the end of the stmt.

syntax:
------
	try()
	{
		- // risky code
	}

	{
		- //error handling code 
	}

java.io package
================

File 
=====
It checks is there any "abc.txt" file already exist or not. If it is exist then 'f' simply refers to that file. If not exist then it won't create a physical file.

ex:
	File f = new File("abc.txt");


ex:
---
import java.io.File;
class Test 
{
	public static void main(String[] args)
	{
		File f = new File("abc.txt");
		System.out.println(f.exists());//false 
	}
}

A File object is used to create a physical file.

ex:
---
import java.io.*;
class Test 
{
	public static void main(String[] args)
	{
		try
		{
			File f = new File("abc.txt");
			System.out.println(f.exists());//false
		
			f.createNewFile();
			System.out.println(f.exists());//true 
		}
		catch (IOException ioe)
		{
			ioe.printStackTrace();
		}
	}
}
